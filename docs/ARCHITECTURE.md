# 架构设计文档 (Architecture Design Document)

## 1. 系统概述 (System Overview)

### 1.1 项目愿景
**AI Novel Writer** 是一个面向专业创作者的 AI 辅助小说创作平台。我们的核心目标是从传统的"以数据管理为中心"转向**"以创作为中心，AI 为副驾驶"**的沉浸式体验。

### 1.2 核心理念
- **去 CRUD 化**：隐藏枯燥的增删改查表格，用自然语言、卡片式交互和上下文感知的 UI 替代。
- **AI 副驾驶 (Copilot)**：AI 不仅仅是聊天机器人，而是深度集成在写作流程中的助手，能实时感知光标位置、当前章节内容、活跃角色状态，并提供主动建议。
- **沉浸式创作 (Immersive Studio)**：采用类似 VS Code 或 Notion 的单页应用 (SPA) 体验，实现大纲、正文、设定之间的无缝流转。
- **世界观演化**：世界观不再是静态的百科全书，而是随着剧情发展动态演化、有版本记录的活体系统。

### 1.3 适用场景
- **长篇连载**：支持 10 万字以上的长篇网络小说创作，解决长文连贯性问题。
- **多题材支持**：针对修仙、玄幻、科幻、都市等不同题材动态调整 AI 上下文权重。
- **辅助构建**：从零开始构建大纲、人设和世界观，或在写作过程中动态补充设定。

| 层级 | 技术选型 | 版本 | 理由 |

| **数据库 ORM** | Prisma | 7.x | 优秀的类型推断和迁移工具 |

| **AI 模型** | Gemini | 2.5 | 长上下文窗口 (1M tokens) 适合长篇小说处理 |

## 3. UI/UX 设计 (User Interface & Experience)

### 3.1 核心布局：Studio 三栏式设计

摒弃传统的顶部导航 + 页面路由模式，采用类似 IDE 的**单页应用 (SPA)** 布局。

```
┌───┬──────────────────────────────────┬───────────────────┐
│ N │                                  │                   │
│ a │                                  │                   │
│ v │           主创作区                │    智能副驾驶      │
│   │         (Main Canvas)            │    (AI Copilot)   │
│   │                                  │                   │
│   │                                  │                   │
└───┴──────────────────────────────────┴───────────────────┘
```

#### 1. 左侧：导航与资源 (Navigation & Resources)
*   **宽度**：固定 250px (可折叠)
*   **视觉**：深色背景 (muted/5)，清晰的层级缩进，图标辅助识别。
*   **功能**：
    *   **工作台**：项目概览卡片，显示封面、字数进度条。
    *   **资源树**：统一展示大纲树、章节列表、角色库、世界观条目。支持拖拽排序。
    *   **过滤器**：快速筛选角色类型、世界观分类。

#### 2. 中间：主创作区 (Main Canvas)
*   **宽度**：自适应，占据剩余空间
*   **视觉**：纯净背景，类似纸张的阅读体验。去除不必要的边框和干扰元素。
*   **功能**：
    *   **顶栏**：面包屑导航，章节标题编辑，保存状态指示器，AI 续写/润色快捷按钮。
    *   **沉浸式编辑器**：基于 TipTap，支持 Markdown，专注于写作本身。
        *   **选中菜单**：选中文字后弹出 Bubble Menu (AI 重写、纠错、提取实体)。
        *   **斜杠命令**：输入 `/` 呼出 AI 指令菜单 (续写段落、插入描写、生成对话)。
    *   **多模式切换**：右上角切换“写作模式”（纯文本）、“大纲模式”（思维导图/节点图）。

#### 3. 右侧：智能副驾驶 (AI Copilot)
*   **宽度**：固定 350px (可折叠)
*   **视觉**：独立的面板区域，卡片式内容展示，强调流式生成的动态感。
*   **Tabs**：
    *   **对话 (Chat)**：Agent 模式。
        *   **操作指令**：“创建一个叫萧炎的角色” -> 直接在数据库创建并返回卡片。
        *   **原文引用**：AI 回答可点击跳转到正文对应位置。
    *   **上下文 (Context)**：Context Radar。
        *   **实时感知**：根据左侧光标所在的段落，自动显示当前出场的角色卡片（状态、好感度）。
        *   **伏笔提醒**：检测到相关关键词时，高亮显示未回收的伏笔。
    *   **工具箱 (Generators)**：表单式生成器。
        *   批量生成路人 NPC、道具、功法、地名。

### 3.2 视觉风格与反馈 (Visual Identity & Feedback)

*   **风格**：**Neo-Brutalism (新野兽派)** 与 **Clean UI** 的结合。
    *   **字体**：使用系统字体栈 (Inter/San Francisco/Segoe UI) 保证清晰度。
    *   **圆角**：适度的圆角 (radius-md)，兼顾现代感与硬朗感。
*   **色彩**：
    *   **主题**：默认 **深色模式 (Dark Mode)**，降低屏幕亮度对眼睛的刺激。
    *   **强调色**：使用 **霓虹紫/蓝** (Violet/Blue) 作为主色调，体现 AI 的智能感。
    *   **状态色**：
        *   Success: 绿色 (Emerald)
        *   Warning: 黄色 (Amber)
        *   Error: 红色 (Rose)
*   **动效与反馈**：
    *   **流式响应**：AI 生成内容时，光标处有闪烁动效，文字逐字上屏，模拟打字机效果。
    *   **加载状态**：使用骨架屏 (Skeleton) 代替转圈 Loading，减少等待焦虑。
    *   **操作反馈**：关键操作（保存、创建、删除）使用 Toast 轻提示。
    *   **面板过渡**：侧边栏折叠/展开使用平滑的 CSS Transition。

### 3.3 核心交互流程细节 (Interaction Details)

#### 1. 启动与构思 (Onboarding)
*   **引导式对话**：首次创建项目时，不直接进入空列表。而是通过全屏对话框询问：“你想写个什么故事？”
*   **AI 脑暴**：用户输入简单想法 -> AI 推荐 3 个创意方向（包括核心冲突、结局走向） -> 用户选择或修改 -> 系统自动生成初始大纲和核心世界观。

#### 2. 角色创建与管理 (Character Management)
*   **拖拽式创建**：在右侧 Copilot 对话中，AI 生成的角色卡片可以直接**拖拽**到左侧“角色库”中保存。
*   **正文提取**：在写作时，选中一段对新人物的描写 -> 悬浮菜单选择“提取为新角色” -> AI 自动分析文本，提取姓名、外貌、性格填入角色表单 -> 用户确认保存。

#### 3. 智能写作辅助 (AI-Assisted Writing)
*   **幽灵文本 (Ghost Text)**：当用户停止输入超过 3 秒，AI 根据上下文预测后续内容，以浅灰色文字显示在光标后。用户按 `Tab` 键采纳，按 `Esc` 键忽略。
*   **局部重绘 (In-painting)**：用户框选一段不满意的描写 -> 点击“AI 重写” -> 选择风格（更黑暗、更幽默、更简练） -> AI 原地替换文本，并保留撤销历史。
*   **实时一致性检查**：如果用户写的内容与设定冲突（例如：角色使用了未学会的技能），右侧 Context 面板会弹出黄色警告卡片，提示“检测到设定冲突”。
